# njX UI Library — Roadmap улучшений

> Документ описывает что нужно сделать, чтобы библиотека стала удобной для внешних пользователей,
> а сайт-документация был поддерживаемым и масштабируемым.

---

## 1. Технологический стек — React vs Alpine.js

### Текущее состояние
Сайт написан на ванильном HTML + JS. Всё в одном `index.html` (~5000 строк). Поддерживать руками тяжело: добавление нового компонента требует копировать шаблон вручную, переводы жить нигде не будут, динамика ограничена.

### Варианты

#### Вариант A — Alpine.js (рекомендуется)
**Плюсы:**
- Минимальная переработка: HTML остаётся HTML, добавляются `x-data`, `x-show`, `x-text`
- Нет сборки, нет node_modules в проде, открываем как статику
- Размер 15 KB — не влияет на скорость
- Идеально для i18n: `x-text="t('button.primary')"` и словарь в JS объекте
- Компоненты секций можно вынести в отдельные HTML-фрагменты и подтягивать через fetch
- Переход займёт 2–3 дня, не нужно переписывать CSS

**Минусы:**
- Нет компонентной системы как в React — при сложном UI придётся городить

**Когда выбрать:** если приоритет — быстро запустить, минимум зависимостей, статический хостинг.

#### Вариант B — React (Vite + React)
**Плюсы:**
- Полноценные компоненты: `<ComponentDemo name="Cards" variants={cardVariants} />`
- i18n через `react-i18next` — стандарт
- Лёгкий рефакторинг: каждая секция — отдельный файл
- Storybook можно подключить прямо в проект

**Минусы:**
- Нужна сборка (Vite), node_modules
- Переписать с нуля займёт 1–2 недели
- Избыточен, если сайт остаётся документацией без интерактивных playground-ов

**Когда выбрать:** если планируется live-редактор кода, playground для компонентов, или публичный npm-пакет.

### Итоговая рекомендация
**Alpine.js** — оптимальный выбор для текущего масштаба.
**React** — если в roadmap есть live-playground или публикация на npm.

## 2. Что нужно для пользователей библиотеки

### 2.1 Установка
- [ ] Добавить `package.json` → публикация на npm как `njx-ui`

- [ ] Отдельный `style.min.css` (минифицированный) для продакшена

### 2.2 Документация компонентов
- [ ] Каждый компонент должен иметь: описание, пропсы/классы, пример, notes (edge-cases)
- [ ] Раздел "Кастомизация" — как переопределить токены через `:root`

- [ ] Changelog (`check.html` уже есть — нужно регулярно вести)

## 3. Что нужно для поддержки сайта

### 3.1 Структура файлов (сейчас vs цель)

```
Сейчас:                          Цель:
index.html (5000 строк)    →     src/
                                 ├── components/
                                 │   ├── buttons.html
                                 │   ├── cards.html
                                 │   ├── forms.html
                                 │   └── ... (1 файл = 1 секция)
                                 ├── i18n/
                                 │   ├── ru.json
                                 │   ├── en.json
                                 │   └── uk.json
                                 ├── js/
                                 │   ├── app.js
                                 │   ├── i18n.js
                                 │   └── download.js
                                 └── index.html (< 200 строк — только shell)
```

### 3.2 Добавление нового компонента (целевой процесс)
1. Создать файл `src/components/новый-компонент.html`
2. Добавить ключи переводов в `i18n/ru.json`, `i18n/en.json`, `i18n/uk.json`
3. Зарегистрировать в навигации (один массив в `app.js`)
4. Готово — никаких изменений в `index.html`

### 3.3 Версионирование
- [ ] Семантическое версионирование: `MAJOR.MINOR.PATCH`
- [ ] `CHANGELOG.md` с датами и описанием изменений
- [ ] Git теги на релизы: `git tag v1.1.0`

## 4. Интернационализация (i18n)

### Языки
| Язык       | Код | Статус        |
|------------|-----|---------------|
| Русский    | ru  | ✅ текущий    |
| English    | en  | 🔲 нужно      |
| Українська | uk  | 🔲 нужно      |

### Что переводить
- UI сайта: заголовки секций, описания, подсказки, кнопки
- Комментарии в примерах кода (`/* _buttons.css — основные */`)
- Описания компонентов в doc.html
- Нотификации (копирование, скачивание)

**Что НЕ переводить:**
- Названия CSS-классов (`.btn-primary` — универсальны)
- Примеры кода — оставить на английском
- Названия файлов

### Реализация на Alpine.js

```html
<!-- index.html -->
<html x-data="app()" :lang="locale">

<!-- Переключатель языка в хедере -->
<div class="lang-switcher">
  <button @click="setLocale('ru')" :class="{ active: locale === 'ru' }">RU</button>
  <button @click="setLocale('en')" :class="{ active: locale === 'en' }">EN</button>
  <button @click="setLocale('uk')" :class="{ active: locale === 'uk' }">UA</button>
</div>

<!-- Использование перевода -->
<div class="lib-section-title" x-text="t('sections.buttons.title')"></div>
<div class="lib-section-desc" x-text="t('sections.buttons.desc')"></div>
```

```js
// i18n/ru.json
{
  "sections": {
    "buttons": {
      "title": "Кнопки",
      "desc": "Варианты из _buttons.css"
    }
  },
  "ui": {
    "copy": "Копировать",
    "copied": "Скопировано!",
    "download": "Скачать CSS",
    "downloading": "Упаковка..."
  }
}
```

### Переключатель языка в хедере
Добавить рядом с темами — три кнопки: `RU | EN | UA`.
Язык сохранять в `localStorage`. При первом открытии определять через `navigator.language`.

## 5. Динамика и функциональность

### 5.1 Быстрые wins (без смены стека)
- [ ] **Поиск по всем компонентам** — одна строка поиска фильтрует весь index
- [ ] **Anchor highlight в sidebar** — текущая секция выделяется при скролле (IntersectionObserver)
- [ ] **Горячие клавиши** — `Cmd+K` открывает поиск, `C` копирует последний код
- [ ] **Сохранение темы** — уже есть? Проверить сохранение в localStorage
- [ ] **Print/Export** — кнопка печати документации секции

### 5.2 Средний приоритет
- [ ] **Live code editor** — textarea с кодом, рядом iframe-превью
- [ ] **Сравнение тем** — checkbox "показать в N темах"
- [ ] **Избранное** — пометить компоненты звёздочкой, сохранить в localStorage
- [ ] **История изменений** — раздел с diff между версиями

### 5.3 Долгосрочно
- [ ] **npm пакет** — `npm install njx-ui`
- [ ] **Figma kit** — компоненты в Figma, синхронизированные с токенами
- [ ] **VS Code extension** — автодополнение классов
- [ ] **Storybook** — если переедем на React
